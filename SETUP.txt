REQUIREMENTS
------------
• PHP 7.4+ with PDO and PDO_MySQL extensions
• MySQL 5.7+ or MariaDB 10.3+
• Apache with mod_rewrite enabled (WAMP, XAMPP, or live server)
• A Google Maps API key (for customer delivery tracking)
• Composer is optional but recommended for worker scripts


STEP 1 — DATABASE SETUP
------------------------
1. Open phpMyAdmin (or your MySQL client).
2. Import database/schema.sql — this creates the database,
   application tables, realistic sample data, and default
   demo accounts.

Default login credentials (all use same password format):
  Admin     : username=admin         password=Admin@1234
              → frontend/admin/index.html

  Outlet staff accounts:
    Sefalana Bakery    : sefalana_staff / Staff@1234
    Blue White Plate   : bwp_manager    / Staff@1234
    Hot Spot Food Court: hotspot_staff  / Staff@1234
              → frontend/dashboard/index.html

  Customers: register via the customer app signup form.
              → frontend/customer/index.html


STEP 2 — BACKEND CONFIGURATION
--------------------------------
Copy .env.example to .env in the project root and set:
  DB_HOST, DB_USER, DB_PASS, DB_NAME
  JWT_SECRET       — use at least 32 random characters
  CORS_ALLOWED_ORIGINS — comma-separated allowed frontend origins
  PAYMENT_PROVIDER=sandbox for local testing
  PAYMENT_PROVIDER_ENABLED=false until a real gateway is integrated
  EMAIL_PROVIDER, SMS_PROVIDER, PUSH_PROVIDER keys before production

Do not commit real .env values or production secrets.


STEP 3 — GOOGLE MAPS API KEY
------------------------------
In frontend/customer/index.html, set:
  window.APP_CONFIG = { googleMapsApiKey: 'YOUR_KEY' };

Use a restricted browser key from:
  https://console.cloud.google.com/
Required APIs: Maps JavaScript API, Geocoding API.

Without a key, maps will not render but all other features work.


STEP 4 — FILE PATHS
---------------------
The frontend pages use relative paths to reach the backend.
If you serve the entire project from your web root (e.g.
C:\wamp\www\student-food-app\), the relative paths work as-is.

The default API base is ../../backend/api from each frontend page.
This works when the project is served through Apache as:
  http://localhost/student-food-app/frontend/...

If deploying elsewhere, set window.APP_CONFIG.apiBase before the
frontend scripts load. Do not open the app with file:// because PHP
and backend/.htaccess routing will not run.


STEP 5 — APACHE / MOD_REWRITE
--------------------------------
backend/.htaccess handles URL routing for the PHP API.
Ensure AllowOverride All is set in your Apache config.
On WAMP: click WAMP tray → Apache → Apache modules → rewrite_module.


STEP 6 — BACKGROUND WORKER
---------------------------
Run the DB-backed notification worker from cron, Supervisor, or
your deployment platform:

  composer run worker:notifications

In development it records sandbox deliveries. In production, replace
the sandbox branch in backend/workers/notification_worker.php with
SendGrid/Mailgun/SES, Orange/Twilio/Infobip SMS, and FCM/APNs adapters.

  backend/api/auth/login.php   — Login (customer | staff | admin)
  backend/api/outlets.php      — List/get food outlets with ratings
  backend/api/menu.php         — Get menu items (with category filter)
  backend/api/orders.php       — Place, view, update, cancel orders

  backend/api/refunds.php      — Admin refund ledger/workflow
  backend/api/dispatch.php     — Driver assignment and customer tracking
  backend/api/drivers.php      — Driver availability, GPS, earnings
  backend/api/legal.php        — Legal documents, consent, data requests
  backend/api/vendor_applications.php — Vendor onboarding workflow
  backend/api/driver_applications.php — Driver onboarding workflow
  backend/api/ratings.php      — Submit and view customer ratings
  backend/api/notifications.php— Customer notifications and preferences
  backend/api/devices.php      — Push device registration
  backend/api/admin.php        — Admin: users, staff, reports, ops ledgers
  backend/api/menu_manage.php  — Staff: add/edit/toggle menu items

  frontend/dashboard/index.html— Outlet staff dashboard (SPA)

ORDER STATUS FLOW
------------------
  pending_vendor → accepted → preparing → ready_for_pickup
        ↘ declined_by_vendor              ↓
                                driver_assigned → picked_up
                                                  ↓
                                delivered_pending_confirmation
                                                  ↓
                                             completed

Customers can cancel only while the order is waiting for vendor
confirmation. Vendors explicitly accept or decline each order.
Drivers can accept deliveries only when approved, online, and not
already assigned to an active delivery. Delivery completion requires
customer receipt confirmation before the driver becomes available again.

If you previously imported the database, run:
  database/migrations/2026_05_marketplace_lifecycle.sql

Fresh installations only need database/schema.sql.

COMMON API 404 FIX
-------------------
If the frontend shows:
  API returned 404. Check that Apache is serving backend/.htaccess
  routes and API_BASE is correct

Open the app through Apache, not file:// or a static server:
  http://localhost/student-food-app/frontend/customer/index.html

Then confirm:
  - Apache mod_rewrite is enabled
  - AllowOverride All is set for the project directory
  - backend/.htaccess is present
  - window.APP_CONFIG.apiBase resolves to ../../backend/api or an
    absolute URL for your deployment
